# 人狼ゲーム レギュレーション仕様書

## 1. 概要

本ドキュメントは人狼ゲームの基本ルールと、カスタマイズ可能なレギュレーションオプションについて定義します。様々なルールバリエーションに対応可能な柔軟な仕様となっています。

## 2. 基本ゲーム構造

人狼ゲームは村人陣営と人狼陣営（および第三陣営）が、交互に訪れる昼と夜のフェーズを通じて勝利を目指すゲームです。

### 2.1 ゲームの目的

- **村人陣営**: すべての人狼を排除する
- **人狼陣営**: 人狼の数が村人陣営の生存者数以上になる
- **第三陣営**: 役職固有の勝利条件を満たす

### 2.2 基本的なゲームの流れ

1. プレイヤーに役職が割り当てられる
2. 夜のフェーズ: 各役職が特殊能力を使用
3. 昼のフェーズ: 議論
4. 投票フェーズ:投票によって追放者を決定する
5. 勝利条件判定
6. 2-4を繰り返す

## 3. レギュレーションパラメータ

### 3.1 役職と人数設定

#### 3.1.1 配役

人狼ゲームに登場する役職は以下の通りです：

**村人陣営**
- 村人: 特殊能力なし
- 占い師: 夜に1人を占い、人狼かどうかを知る
- 霊媒師: 処刑された人が人狼かどうかを知る
- 騎士(ガード): 夜に1人を護衛し、人狼の襲撃から守る
- 共有者: お互いが共有者であることを知っている村人

**人狼陣営**
- 人狼: 夜に1人を襲撃できる
- 狂人: 正体は村人だが、人狼陣営の勝利が自身の勝利となる

**第三陣営**
- 妖狐: 人狼に襲撃されても死なないが、占われると死亡する
- 背徳者: 妖狐を信仰する村人。妖狐の生存が自身の勝利条件となる
- その他の独立役職

#### 3.1.2 人数設定

- 最小プレイヤー数: 通常3人以上
- 最大プレイヤー数: 制限なし（推奨は16人程度まで）
- 役職配分: ゲームバランスに応じて調整可能

### 3.2 アクション制限設定

#### 3.2.1 連続ガードの可否

- **許可**: 同じ対象を連続して護衛できる
- **禁止**: 同じ対象を連続して護衛できない（デフォルト）

#### 3.2.2 連続襲撃の可否

- **許可**: 同じ対象を連続して襲撃できる（デフォルト）
- **禁止**: 同じ対象を連続して襲撃できない

#### 3.2.3 自己対象アクションの可否

- **自己護衛**: 騎士が自分自身を護衛できるか
- **自己襲撃**: 人狼が自分自身を襲撃対象に選べるか
- **襲撃なし**: 人狼が襲撃をスキップできるか

#### 3.2.4 初日占いの設定

- **自由占い**: 占い師が自由に対象を選べる（デフォルト）
- **ランダム白**: 初日の占い結果は必ず「村人」になる
- **ランダム結果**: 初日の占い先はランダムで決まる
- **呪殺なし**: 初日の占いでは妖狐が呪殺されない

### 3.3 投票システム設定

#### 3.3.1 投票方式

- **一斉投票**: 全員が同時に投票（デフォルト）
- **順番投票**: プレイヤーが順番に投票する
- **挙手制**: 指名された候補に対して挙手で投票

#### 3.3.2 自投票の可否

- **許可**: 自分自身に投票できる
- **禁止**: 自分自身に投票できない（デフォルト）

#### 3.3.3 同数得票時の処理

- **処刑なし**: 誰も処刑されない
- **ランダム処刑**: 同票者からランダムで1人が処刑（デフォルト）
- **引き分け処理**: 特別なルール（例: 前日処刑者が決定）
- **両方処刑**: 同票の全プレイヤーが処刑される

#### 3.3.4 再投票と決選投票

- **決選投票あり**: 同数得票時に上位得票者のみで再投票
- **決選投票なし**: 同数票の場合は直接同数処理を適用
- **再投票回数**: 最大何回まで再投票を行うか（0〜無制限）

## 4. 特殊役職の詳細

### 4.1 背徳者（Heretic）

背徳者は、妖狐を信仰する村人の役職です。以下の特性を持ちます：

- **陣営**: 妖狐陣営（第三陣営）
- **能力**: 特殊能力はなし
- **勝利条件**: 妖狐の生存。妖狐が全滅した場合は敗北となる
- **判定**: 占い/霊媒結果は「村人」と表示される
- **特記事項**:
  - 妖狐が誰かを知ることはできない（妖狐側からも背徳者が誰か分からない）
  - 妖狐が呪殺されても、背徳者自身は呪殺されない
  - レギュレーションによっては、妖狐が死亡した場合に村人陣営に鞍替えするバリエーションも存在する

### 4.2 レギュレーションにおける背徳者の扱い

背徳者をゲームに導入する際の考慮点：

- **難易度**: 背徳者の存在はゲームの複雑さを増加させる
- **バランス**: 妖狐+背徳者の組み合わせは第三陣営を強化する
- **人数**: 一般的に妖狐1人に対して背徳者は0-1人が推奨される
- **情報共有**: 背徳者と妖狐の間の情報共有ルールを明確にする必要がある

## 5. 標準レギュレーション例

### 5.1 スタンダードルール
- 連続ガード: 禁止
- 自投票: 禁止
- 同数処理: ランダム処刑
- 初日占い: 自由選択
- 投票方式: 一斉投票
- 背徳者: なし

### 5.2 初心者向けルール
- 連続ガード: 許可
- 自投票: 禁止
- 同数処理: 処刑なし
- 初日占い: ランダム白
- 投票方式: 挙手制
- 背徳者: なし

### 5.3 上級者向けルール
- 連続ガード: 禁止
- 自投票: 禁止
- 同数処理: 両方処刑
- 初日占い: 自由選択
- 投票方式: 順番投票
- 再投票: なし
- 背徳者: あり（妖狐と組み合わせて使用）

## 6. レギュレーション選択の考慮点

レギュレーションを選択または設計する際は、以下の点を考慮することが重要です：

### 6.1 プレイヤー経験
- 初心者プレイヤーには単純なルールが適している
- 上級者には戦略の幅を広げるルールが好まれる

### 6.2 ゲームバランス
- 村人陣営と人狼陣営の勝率バランス
- 各役職の有用性と影響力
- 第三陣営（妖狐・背徳者）の強さの調整

### 6.3 ゲーム時間
- 厳格なルールほどゲーム時間が長くなる傾向
- 再投票や決選投票はゲーム時間を延長する

### 6.4 戦略の多様性
- ルールの複雑さと戦略の幅はトレードオフの関係
- カスタムレギュレーションで新たな戦略を生み出すことが可能

## 7. カスタムレギュレーション設計ガイドライン

独自のレギュレーションを設計する際は、以下のガイドラインに従うことを推奨します：

1. **一貫性**: 互いに矛盾するルールを避ける
2. **バランス**: どちらの陣営にも勝機がある状態を維持する
3. **明確さ**: すべてのプレイヤーが理解できるシンプルな説明
4. **テスト**: 実際のプレイで検証し調整を行う
